            # 全称量词

            sql语句中没有全称量词，但是可以用存在量词实现同样功能。

            所有人都吃了饭 <=> 没有人-没有吃饭

            1. 查询选了所有课程的人 <=> 没有课程-该学生没有选
            
select *
from student
where not exists(
  select *
  from course
  where not exists(
    select *
    from sc
    -- 选课学生
    where student.sno = sc.sno
    -- 选择的课程
    and sc.cno=course.cno
  )
)

            # 连接条件

            谓词连接使用where，过滤条件一样。

            内连接单独使用on，过滤条件使用where。

            # 多表连接

            多表连接时，注意连接条件按顺序。

            中间表（公共属性）的连接条件应该写在前面

            如：查询结果包含学生姓名(Student)、选课分数(SC)、选课名(Course)

            SELECT Sname, Grade, Cname
FROM Student join SC join Course
-- 此时该表已正常连接
ON SC.cno=Course.Cno
ON Student.Sno=Sc.Sno

            # 索引

            便于搜索，提高查询性能。效果显著。

            一个好的数据库设计，一定要有好的索引。

            索引分为聚集索引、非聚集索引

            聚合索引排序，且唯一。

            一张表内只能有一个聚合索引，可以有多个非聚合索引。

            列名添加PRIMARY KEY约束时，该列默认拥有UNIQUE、NOT NULL、INDEX属性。

            语法

            CREATE UNIQUE INDEX 索引名 ON [表名 | 视图名] (列名);

CREATE UNIQUE INDEX unq_name ON Student (Sname);

            # 视图

            相当于一个函数、过滤器，或是子表。完全可以当做一个表进行操作，也可以增删改查。

            子查询不建议使用*，应指定好列名。

            例子：创建视图查询所有计算机系的学生

            CREATE VIEW CS_STUDENT(Sno, Sname, Sage, Ssex) -- 可选列
AS
SELECT Sno, Sname, Sage, Ssex
FROM Student
WHERE Sdept='CS'
